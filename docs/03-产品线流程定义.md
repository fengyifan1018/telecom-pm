# 通信项目任务管理系统 - 产品线流程定义

## 1. 传输产品（MSTP/OTN）

### 标准交付流程

```
阶段1: 项目立项          角色: 项目经理
  ├─ 项目信息录入确认
  ├─ 客户需求澄清
  └─ 交付团队组建

阶段2: 现场勘察          角色: 现场实施人员
  ├─ 客户侧机房环境确认
  ├─ 传输路由方案确认
  └─ 勘察报告提交

阶段3: 方案设计          角色: 网络工程师          依赖: 阶段2
  ├─ 传输通道设计
  ├─ 端口/时隙规划
  ├─ 网管配置方案
  └─ 方案评审

阶段4: 资源确认          角色: 运营              依赖: 阶段3
  ├─ 传输通道资源核实
  ├─ 跨运营商资源协调（如有）
  └─ 资源锁定确认

阶段5: 采购              角色: 采购              依赖: 阶段3（可与阶段4并行）
  ├─ 设备需求确认
  ├─ 供应商询价/下单
  ├─ 到货验收
  └─ 发货至站点

阶段6: 施工实施          角色: 现场实施人员       依赖: 阶段4 + 阶段5
  ├─ 设备上架安装
  ├─ 光路/电路接入
  ├─ 传输配置开通
  └─ 环路测试

阶段7: 联调验收          角色: 网络工程师 + 现场   依赖: 阶段6
  ├─ 端到端业务验证
  ├─ 性能指标测试
  ├─ 客户验收签字
  └─ 验收报告提交

阶段8: 项目结项          角色: 运营/项目经理       依赖: 阶段7
  ├─ 文档归档
  ├─ 运维交接
  └─ 项目关闭
```

### 典型交付周期：15-45 工作日

## 2. 裸纤产品

### 标准交付流程

```
阶段1: 项目立项          角色: 项目经理
  ├─ 项目信息录入
  ├─ A/Z端地址确认
  └─ 初步路由评估

阶段2: 路由勘察          角色: 现场实施人员
  ├─ A端机房勘察
  ├─ Z端机房勘察
  ├─ 中间路由勘察
  ├─ 管道/杆路资源确认
  └─ 勘察报告 + 路由图

阶段3: 资源协调          角色: 运营              依赖: 阶段2
  ├─ 光缆资源确认
  ├─ 管道使用许可申请
  ├─ 占道/施工许可（如需）
  └─ 资源费用确认

阶段4: 施工方案          角色: 网络工程师         依赖: 阶段2
  ├─ 光缆敷设方案
  ├─ 熔接方案
  ├─ 施工排期
  └─ 安全方案

阶段5: 施工实施          角色: 现场实施人员       依赖: 阶段3 + 阶段4
  ├─ 光缆敷设
  ├─ 光纤熔接
  ├─ ODF跳纤
  └─ 施工记录

阶段6: 测试验收          角色: 网络工程师 + 现场   依赖: 阶段5
  ├─ OTDR测试
  ├─ 光功率测试
  ├─ 测试报告
  └─ 客户验收签字

阶段7: 项目结项          角色: 运营/项目经理       依赖: 阶段6
  ├─ 纤芯标识录入资源系统
  ├─ 文档归档
  └─ 项目关闭
```

### 典型交付周期：10-60 工作日（取决于是否有现成资源）

## 3. DIA 产品（专线互联网接入）

### 标准交付流程

```
阶段1: 项目立项          角色: 项目经理
  ├─ 项目信息录入
  ├─ 带宽/IP需求确认
  └─ 接入方式确认（光纤/铜缆）

阶段2: 资源确认          角色: 运营              依赖: 阶段1
  ├─ 末端接入资源确认
  ├─ 上游运营商通道确认
  ├─ IP地址分配
  └─ 端口预留

阶段3: 采购/协调         角色: 采购              依赖: 阶段2（如需新设备）
  ├─ CPE设备确认
  ├─ 光模块采购（如需）
  └─ 发货安排

阶段4: 施工开通          角色: 现场实施人员       依赖: 阶段2 + 阶段3
  ├─ 末端线路接入
  ├─ CPE安装配置
  ├─ 端口开通
  └─ 基础连通测试

阶段5: 网络配置          角色: 网络工程师         依赖: 阶段4
  ├─ 路由配置
  ├─ 带宽策略配置
  ├─ ACL/安全策略
  └─ 网管纳管

阶段6: 验收              角色: 网络工程师         依赖: 阶段5
  ├─ 带宽测试
  ├─ 延迟/丢包测试
  ├─ 冗余切换测试（如有）
  ├─ 客户验收确认
  └─ 验收报告

阶段7: 项目结项          角色: 运营               依赖: 阶段6
  ├─ 资源系统录入
  ├─ 监控告警配置
  └─ 项目关闭
```

### 典型交付周期：5-15 工作日

## 4. SD-WAN 产品

### 标准交付流程

```
阶段1: 项目立项          角色: 项目经理
  ├─ 项目信息录入
  ├─ 站点清单确认
  ├─ 带宽/线路需求确认
  └─ 组网拓扑确认

阶段2: 方案设计          角色: 网络工程师         依赖: 阶段1
  ├─ 组网方案设计
  ├─ CPE选型
  ├─ 策略规划（QoS/安全/路由）
  ├─ 控制器部署方案
  └─ 方案评审

阶段3: 采购              角色: 采购              依赖: 阶段2
  ├─ CPE设备采购
  ├─ License采购
  ├─ 设备预配置（ZTP模板）
  └─ 分站点发货

阶段4: 底层线路          角色: 运营/现场          依赖: 阶段1（可与阶段2并行）
  ├─ 各站点Underlay线路开通
  ├─ 线路测试
  └─ 线路信息登记

阶段5: 站点部署          角色: 现场实施人员       依赖: 阶段3 + 阶段4
  ├─ CPE安装上电
  ├─ 线路对接
  ├─ 设备注册上线
  └─ 各站点逐一开通

    注: 多站点可并行，系统支持子任务分站点跟踪

阶段6: 策略配置          角色: 网络工程师         依赖: 阶段5（至少Hub站完成）
  ├─ Overlay隧道建立
  ├─ 路由策略下发
  ├─ QoS策略下发
  ├─ 安全策略下发
  └─ 应用识别策略

阶段7: 联调验收          角色: 网络工程师         依赖: 阶段5全部 + 阶段6
  ├─ 端到端连通验证
  ├─ 策略生效验证
  ├─ 故障切换测试
  ├─ 性能基线测试
  ├─ 客户UAT
  └─ 验收报告

阶段8: 项目结项          角色: 运营/项目经理       依赖: 阶段7
  ├─ 平台监控配置
  ├─ 运维文档移交
  ├─ 客户培训
  └─ 项目关闭
```

### 典型交付周期：15-30 工作日（视站点数量）

### SD-WAN 多站点并行管理

SD-WAN项目特殊性：一个项目包含多个站点，各站点独立推进但整体关联。

数据模型扩展：
```sql
CREATE TABLE project_site (
    id              BIGSERIAL PRIMARY KEY,
    project_id      BIGINT NOT NULL REFERENCES project(id),
    site_name       VARCHAR(100) NOT NULL,
    site_address    VARCHAR(300),
    site_type       VARCHAR(20),          -- hub/spoke
    contact_name    VARCHAR(50),
    contact_phone   VARCHAR(20),
    status          VARCHAR(20) DEFAULT 'pending',
    created_at      TIMESTAMP DEFAULT NOW()
);

-- 任务表增加站点关联
ALTER TABLE task ADD COLUMN site_id BIGINT REFERENCES project_site(id);
```

## 5. 各产品线对比

| 维度 | 传输 | 裸纤 | DIA | SD-WAN |
|------|------|------|-----|--------|
| 标准阶段数 | 8 | 7 | 7 | 8 |
| 核心复杂度 | 跨网协调 | 物理施工 | 上游协调 | 多站点并行 |
| 常见瓶颈 | 资源确认 | 施工许可 | 上游开通 | CPE到货 |
| 并行度 | 中 | 低 | 中 | 高 |
| 外部依赖 | 其他运营商 | 市政/物业 | 上游运营商 | 设备供应商 |
| 典型周期 | 15-45天 | 10-60天 | 5-15天 | 15-30天 |

# 通信项目任务管理系统 - 需求分析

## 1. 业务背景

通信行业项目交付涉及多产品线（传输、裸纤、DIA、SD-WAN），每个项目从签单到交付需经历多个角色协作。当前痛点：

- 任务在角色之间流转时信息断层（销售签单后，项目经理不知道具体需求细节）
- 各角色使用不同工具（Excel、邮件、微信），状态不统一
- 无法快速定位项目卡在哪个环节、哪个人手上
- 采购和资源协调缺乏提前量，经常临时抱佛脚

## 2. 角色与职责

| 角色 | 核心职责 | 系统中的关注点 |
|------|----------|----------------|
| 销售 | 签单、录入项目信息、提出交付需求 | 项目进度可见、客户交付时间承诺 |
| 项目经理 | 整体协调、拆解任务、推动进度 | 全局视图、瓶颈预警、任务分派 |
| 运营 | 审核资源、商务流程处理 | 审批流、资源库存状态 |
| 采购 | 设备/资源采购 | 采购需求清单、供应商跟进、到货时间 |
| 网络工程师 | 方案设计、网络配置 | 技术方案模板、配置任务 |
| 现场实施人员 | 现场施工、开通验证 | 工单详情、施工规范、验收签字 |

## 3. 产品线特点

### 3.1 传输（MSTP/OTN）
- 流程长：勘察→设计→资源确认→施工→联调→验收
- 涉及多段资源协调（可能跨运营商）
- 关键节点：传输通道确认、端口分配

### 3.2 裸纤
- 重资源：光纤路由确认、管道资源
- 关键节点：路由确认、施工许可、熔接测试
- 周期波动大（取决于是否有现成资源）

### 3.3 DIA（专线互联网接入）
- 相对标准化，流程较短
- 关键节点：IP分配、端口开通、带宽配置
- 涉及上游运营商协调

### 3.4 SD-WAN
- 设备+软件双维度
- 关键节点：CPE设备采购/发货、控制器配置、策略下发
- 涉及多站点并行交付

## 4. 核心业务流程（通用骨架）

```
销售签单 → 项目立项 → 资源/方案设计 → 采购/资源确认 → 实施部署 → 验收交付
```

各阶段任务流转：

```
┌─────────┐     ┌──────────┐     ┌──────────┐     ┌────────┐     ┌──────────┐     ┌────────┐
│  销售    │────→│ 项目经理  │────→│ 网络工程师│────→│  采购   │────→│ 现场实施  │────→│  运营   │
│ 录入项目 │     │ 立项拆解  │     │ 方案设计  │     │ 下单采购│     │ 施工开通  │     │ 验收结项│
└─────────┘     └──────────┘     └──────────┘     └────────┘     └──────────┘     └────────┘
                      │                                                │
                      └────────────── 全程监控、协调推动 ───────────────┘
```

## 5. 功能需求

### 5.1 项目管理（项目经理视角）
- 项目立项：关联产品线、客户、合同
- 任务拆解：按产品线模板自动生成任务清单
- 任务分派：指定负责人、设置截止时间
- 进度看板：甘特图/看板视图
- 瓶颈预警：超时任务自动标红、通知

### 5.2 任务流转引擎（核心）
- 阶段流转规则：前置任务完成后自动触发下一阶段
- 并行任务支持：如采购和方案设计可并行
- 任务状态机：待分配→进行中→待审核→已完成→已关闭
- 流转通知：任务到达时自动通知负责人（站内消息+企微/钉钉）
- 退回机制：任务可退回上一环节并附带说明

### 5.3 协作与沟通
- 任务评论：在任务上下文中讨论，避免信息散落
- 附件管理：方案文档、施工照片、验收报告
- @提醒：在评论中@相关人触发通知
- 操作日志：谁在什么时间做了什么操作

### 5.4 产品线模板
- 预定义各产品线的标准任务模板
- 项目立项时自动生成任务树
- 模板可由运营/项目经理维护

### 5.5 仪表盘与报表
- 项目总览：按状态/产品线/客户分类统计
- 个人工作台：我的待办、我发起的、我关注的
- 交付周期分析：各产品线平均交付天数、卡点分布
- 人员负荷：各角色当前任务量

### 5.6 权限模型
- 基于角色的权限（RBAC）
- 数据范围：销售只看自己的项目，项目经理看所管项目，管理层看全部
- 操作权限：各角色只能操作自己阶段的任务

## 6. 非功能需求

| 维度 | 要求 |
|------|------|
| 并发 | 50-200人同时在线 |
| 响应 | 页面加载 <2s，操作响应 <500ms |
| 可用性 | 99.5%（工作时间） |
| 移动端 | 现场人员需要手机端操作（审批、拍照上传） |
| 集成 | 对接企业微信/钉钉通知，可选对接OA审批 |
| 数据安全 | 客户信息脱敏、操作审计日志 |

## 1. 系统概述

本系统用于管理通信行业项目（DIA专线、传输、裸纤、SD-WAN）从签单到交付的全流程任务协作，解决多角色信息断层、进度不透明的问题。

**核心流程：**

```
销售创建项目 → 项目经理立项（生成任务）→ 分派给各角色 → 执行/提交/审核 → 阶段完成自动解锁下一阶段 → 项目完成
```

## 3. 角色与权限

| 角色 | 标识 | 主要权限 |
|------|------|----------|
| 销售 | sales | 创建项目、查看自己负责的项目和任务、提交变更申请 |
| 项目经理 | pm | 立项、分派任务、审批任务、暂停/恢复项目、审批变更申请 |
| 运营 | operations | 查看所有项目，处理运营类任务 |
| 采购 | procurement | 只能看到分配给自己的任务 |
| 网络工程师 | network_engineer | 只能看到分配给自己的任务 |
| 现场实施 | field_engineer | 只能看到分配给自己的任务 |
| 管理员 | admin | 全部权限，包括删除项目、用户管理、模板管理 |

> **数据隔离说明：**
> - 销售只能看到自己负责的项目及其任务
> - 采购、网络工程师、现场实施只能看到指派给自己的任务
> - 项目经理和管理员可查看所有项目和任务

## 4. 销售：创建项目

### 4.1 新建项目

1. 进入 **项目管理** 页面
2. 点击右上角 **新建项目**
3. 填写以下信息：

   | 字段 | 说明 | 是否必填 |
   |------|------|----------|
   | 项目名称 | 如"XX客户DIA接入" | 必填 |
   | 客户 | 从下拉框选择已有客户，或直接输入新客户名称自动创建 | 必填 |
   | 产品线 | DIA专线 / 传输 / 裸纤 / SD-WAN | 必填 |
   | 计划周期 | 预计开始和结束日期 | 建议填写 |
   | 项目详情 | 项目背景、需求说明、技术要求等 | 建议填写 |

4. 点击 **创建**，项目以"草稿"状态建立，系统自动分配项目编号（如 `PRJ-2026-0001`）

> 销售创建项目后，所指定的项目经理会收到系统通知。

### 4.2 查看项目进度

- 项目列表展示：客户名称、产品线、计划周期、状态、项目经理
- 点击任意行进入**项目详情页**，可查看阶段进度、任务列表
- 提交**变更申请**（见第 7.4 节）

## 5. 项目经理：立项与任务分派

### 5.1 立项（生成任务）

项目处于**草稿**状态时，项目经理需要执行立项操作：

1. 打开项目详情页
2. 点击右上角 **立项（生成任务）**
3. 系统根据该产品线的流程模板自动生成全部阶段任务
4. 立项成功后，项目状态变为**进行中**，首个阶段的任务自动激活

> 立项后销售会收到通知，告知项目已启动并生成了多少任务。

### 5.2 分派任务

立项后需要将任务分配给对应执行人：

1. 在任务列表点击任意任务，打开**任务抽屉**
2. 在抽屉的"指派"区域选择**负责人**和**审核人**
3. 如果被指派人当前进行中任务 ≥ 3 个，会出现黄色工作量预警（不强制阻止）
4. 点击**指派**，对方收到通知

### 5.3 审批任务

当执行人提交任务后，审核人需要审批：

- **通过**：任务变为"已完成"
- **退回**：需填写退回原因，任务退回给执行人重做，`重做次数 +1`

### 5.4 项目生命周期管理

| 操作 | 条件 | 说明 |
|------|------|------|
| 暂停 | 项目进行中 | 需填写原因，进行中的任务自动变为"已暂停" |
| 恢复 | 项目已暂停 | 任务恢复为进行中，**自动顺延**所有任务截止日期（按实际暂停天数） |
| 取消项目 | 项目未完成/未取消 | 不可恢复 |

## 6. 执行人员：处理任务

### 6.1 找到我的任务

有两个入口：
- **我的工作台**：首页，展示本人所有待处理任务
- **任务看板**：按状态列（待处理 / 进行中 / 待审核 / 已完成 / 异常）查看

### 6.2 任务状态流转

```
待处理(pending) → [开始] → 进行中(active) → [提交] → 待审核(review)
                                                           ↓ 审核通过
                                                       已完成(done)
                                                           ↓ 审核退回
                                                       进行中(active)
```

异常状态：
- **已暂停(paused)**：项目暂停时自动触发
- **阻塞(blocked)**：上游任务被退回时自动触发，上游修复后自动解除

### 6.3 操作步骤

1. 点击任务打开**任务抽屉**
2. 查看任务详情（标题、阶段、截止日期、负责人、审核人）
3. 点击 **开始任务** 将状态改为"进行中"
4. 完成工作后点击 **提交审核**（可填写备注说明）
5. 等待审核人操作

### 6.4 评论与@提及

在任务抽屉下方评论框：
- 可输入文字留言，所有相关人员可见
- 支持 `@用户显示名` 格式，被@的人会收到系统通知

## 7. 项目详情页功能详解

### 7.1 头部信息

显示：项目名称、编号、状态、产品线、客户、销售、项目经理、计划周期

状态含义：

| 状态 | 说明 |
|------|------|
| 草稿 | 已创建，待立项 |
| 进行中 | 已立项，任务推进中 |
| 已暂停 | 人工暂停 |
| 已完成 | 所有必要任务完成，系统自动变更 |
| 已取消 | 人工取消 |

### 7.2 项目详情卡片

显示创建时填写的项目背景和需求说明（有内容才显示）。

### 7.3 阶段进度

横向进度条，按阶段展示完成率，一眼看出项目卡在哪个环节。

### 7.4 变更申请（CR）

当项目范围、工期、需求需要变更时：

**提交变更申请（销售/PM/管理员）：**
1. 点击"变更申请" Tab
2. 点击 **提交变更申请**
3. 填写标题、变更原因、详细说明
4. 提交后，项目经理收到通知

**审批变更申请（项目经理/管理员）：**
- 在变更申请列表，对"待审批"的申请点击**批准**或**拒绝**
- 拒绝时需填写原因

CR 状态：待审批 / 已批准 / 已拒绝，均有记录可追溯。

### 7.5 阶段交付物上传

在任务列表每个阶段分组的标题旁：
1. 点击 **+ 上传交付物**
2. 在文件选择框选择文件（支持任意格式，最大 50MB）
3. 弹出标题输入框，填写文件名称后上传
4. 上传成功后，绿色区域显示文件链接，可点击下载

> 删除交付物：只有上传者本人或 PM/管理员可以删除。

### 7.6 导出 Excel

点击任务列表右上角的 **导出 Excel**，下载包含所有任务信息的表格（任务编号、名称、阶段、状态、负责人、计划时间、优先级）。

### 7.7 甘特图

切换到"甘特图"Tab，查看各任务的计划时间轴。

## 8. 任务看板

看板按 5 列展示任务：

| 列 | 状态 | 说明 |
|----|------|------|
| 待处理 | pending | 已分派但未开始 |
| 进行中 | active | 已开始 |
| 待审核 | review | 已提交，等待审核 |
| 已完成 | done | 审核通过 |
| 异常 | blocked/paused | 被阻塞或项目暂停中 |

**拖拽操作：**
- 可将任务卡片拖拽到其他列触发状态流转
- 异常列不支持拖入/拖出

**筛选：**
- 顶部可按项目筛选，查看特定项目的任务看板

## 9. 我的工作台

首页展示当前用户的综合概览：

- **待办任务**：分派给我且状态为"待处理"或"进行中"的任务列表
- **统计数据**：项目总数、进行中任务数、待审核数、已逾期数
- **系统通知**：任务指派、审核结果、@提及、项目状态变化等

通知图标（铃铛）：有未读通知时显示红点，点击查看全部通知。

## 10. 报表

管理员/项目经理可查看统计报表：

### 10.1 项目状态分布
饼图展示各状态（进行中、已完成、已暂停、已取消）项目数量。

### 10.2 各阶段任务状态
横向堆叠柱状图，按阶段查看任务状态分布，一眼识别哪个阶段积压最多。

### 10.3 交付周期分析
各阶段平均停留天数（从开始到完成的实际天数），帮助识别流程瓶颈。

### 10.4 退回率分析
各阶段任务被退回（重做）的比例和平均次数，质量指标参考。

## 11. 流程模板管理

**仅管理员可操作。**

模板定义了每种产品线的标准交付流程，立项时系统根据模板自动生成任务。

### 11.1 查看模板

进入**流程模板管理**，展开任意模板卡片可查看阶段结构和任务列表。

### 11.2 创建/编辑模板（可视化编辑器）

点击**新建模板**或**编辑**按钮，进入可视化编辑器：

**顶部流程条：** 实时预览各阶段顺序，颜色对应负责角色。

**阶段配置（每个阶段卡片）：**

| 字段 | 说明 |
|------|------|
| 阶段名称 | 显示给用户的名称，如"现场施工" |
| 阶段标识 | 英文唯一 key，如 `field_deploy` |
| 负责角色 | 该阶段的主责角色 |
| 预估天数 | 整个阶段预计工期 |
| 前置阶段 | 多选，指定哪些阶段完成后才能激活本阶段（依赖关系） |

**任务配置（每个阶段内）：**

| 字段 | 说明 |
|------|------|
| 任务名称 | 具体任务标题 |
| 必须 | 勾选则为必要任务，所有必要任务完成才触发下一阶段 |
| 预估天数 | 单个任务预计工时 |

**上移/下移：** 调整阶段顺序。

**JSON 预览：** 切换到"JSON 预览"Tab 查看完整配置结构。

### 11.3 停用/启用模板

同一产品线可有多个模板，但同时只有一个启用（`启用中`）。停用后新项目不会使用该模板。

> 修改模板不影响已立项的项目，版本号会自动 +1。

## 13. 常见问题

**Q: 为什么任务显示"阻塞"状态？**  
A: 上游阶段（依赖的阶段）有任务被审核退回时，下游任务自动变为"阻塞"。待上游任务修复并重新完成后，阻塞状态自动解除。

**Q: 项目暂停恢复后，任务截止日期变了？**  
A: 这是正常的。系统会自动按实际暂停天数顺延所有任务的计划结束日期，避免因暂停造成大量任务显示逾期。

**Q: 上传交付物时弹出了文件选择框但选完文件没有上传弹窗？**  
A: 选择文件后会弹出一个标题输入框，输入文件标题（默认为文件名去掉扩展名）后点击确定才会上传。

**Q: 为什么我看不到某个项目/任务？**  
A: 系统按角色隔离数据：销售只能看自己负责的项目；采购/网络工程师/现场实施只能看到指派给自己的任务。如需查看请联系项目经理或管理员。

**Q: 变更申请批准了会发生什么？**  
A: 目前批准/拒绝仅记录状态，实际的计划调整（改日期、改任务）需项目经理手动操作。变更申请作为正式的变更记录留存。

**Q: 模板修改后，已立项的项目任务会变化吗？**  
A: 不会。模板只在立项时使用一次，之后修改模板不影响已生成的任务。
